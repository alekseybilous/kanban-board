## Usage

```css
/* Direct color usage */
color: var(--color-blue-600);
background-color: var(--color-gray-50);

/* Semantic theme variables (recommended) */
color: var(--primary-base);
background-color: var(--surface);
```

## Semantic Color Variables

When building components, prefer using semantic theme variables over direct color tokens:

### Foundation Colors

- `--canvas` - Page background
- `--canvas-subtle` - Subtle page background
- `--surface` - Component background
- `--surface-subtle` - Subtle component background
- `--surface-hover` - Hover state background
- `--surface-active` - Active state background
- `--surface-disabled` - Disabled state background

### Text Colors

- `--text` - Primary text
- `--text-secondary` - Secondary text
- `--text-tertiary` - Tertiary text
- `--text-disabled` - Disabled text
- `--text-inverse` - Text on dark backgrounds
- `--text-link` - Link text
- `--text-link-hover` - Link hover text

### Border Colors

- `--border` - Default border
- `--border-subtle` - Subtle border
- `--border-strong` - Strong border
- `--border-hover` - Hover border
- `--border-focus` - Focus border
- `--border-disabled` - Disabled border

### Interactive Colors

Each semantic color (primary, secondary, success, warning, danger) includes:

- `--{color}-base` - Default state
- `--{color}-hover` - Hover state
- `--{color}-active` - Active state
- `--{color}-disabled` - Disabled state
- `--{color}-surface` - Subtle background
- `--{color}-surface-hover` - Subtle background hover
- `--{color}-border` - Border color
- `--{color}-text` - Text color
- `--{color}-text-on-{color}` - Text on colored background

## Color Accessibility

- Ensure proper contrast ratios between text and background colors
- Use the defined color scale for consistent visual hierarchy
- Test colors across all three themes (light, dark, blue)
- Consider color blindness when using colors to convey meaning
- Always pair color with other visual indicators (icons, text)

## Best Practices

1. **Use semantic variables** - Prefer `--primary-base` over `--color-blue-600`
2. **Follow the scale** - Use predefined shades (50, 100, 200, etc.)
3. **Test across themes** - Ensure your components work in all themes
4. **Maintain contrast** - Use tools to verify accessibility standards
5. **Be consistent** - Use the same color for similar UI elements
