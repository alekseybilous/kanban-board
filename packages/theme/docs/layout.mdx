# Layout

Layout tokens define container sizes, breakpoints, and z-index values for consistent and responsive layouts.

## Container Widths

Predefined maximum widths for content containers and responsive design.

| Token             | Value  | Pixel  | Usage                                |
| ----------------- | ------ | ------ | ------------------------------------ |
| `--container-xs`  | 20rem  | 320px  | Mobile content, narrow forms         |
| `--container-sm`  | 30rem  | 480px  | Small screens, mobile landscape      |
| `--container-md`  | 40rem  | 640px  | Medium content, tablets              |
| `--container-lg`  | 50rem  | 800px  | Large content, readable line lengths |
| `--container-xl`  | 60rem  | 960px  | Extra large content, desktop         |
| `--container-2xl` | 70rem  | 1120px | Wide content layouts                 |
| `--container-3xl` | 80rem  | 1280px | Very wide content                    |
| `--container-4xl` | 90rem  | 1440px | Large desktop layouts                |
| `--container-5xl` | 100rem | 1600px | Extra large layouts                  |
| `--container-6xl` | 110rem | 1760px | Maximum content width                |
| `--container-7xl` | 120rem | 1920px | Full HD layouts                      |

## Responsive Breakpoints

Media query breakpoints for consistent responsive design.

| Token              | Value | Pixel  | Usage                           |
| ------------------ | ----- | ------ | ------------------------------- |
| `--breakpoint-xs`  | 30em  | 480px  | Mobile landscape, small tablets |
| `--breakpoint-sm`  | 40em  | 640px  | Small tablet portrait           |
| `--breakpoint-md`  | 48em  | 768px  | Tablet, small laptop            |
| `--breakpoint-lg`  | 64em  | 1024px | Desktop, large tablet           |
| `--breakpoint-xl`  | 80em  | 1280px | Large desktop                   |
| `--breakpoint-2xl` | 96em  | 1536px | Extra large desktop             |

## Z-Index Scale

Organized layering system for consistent element stacking.

| Token          | Value | Usage                        |
| -------------- | ----- | ---------------------------- |
| `--z-hide`     | -1    | Hidden behind other elements |
| `--z-auto`     | auto  | Browser default stacking     |
| `--z-base`     | 0     | Base layer, normal flow      |
| `--z-docked`   | 10    | Docked elements, toolbars    |
| `--z-dropdown` | 1000  | Dropdown menus               |
| `--z-sticky`   | 1100  | Sticky headers, navigation   |
| `--z-banner`   | 1200  | Notification banners         |
| `--z-overlay`  | 1300  | Overlay backgrounds          |
| `--z-modal`    | 1400  | Modal dialogs                |
| `--z-popover`  | 1500  | Popover content              |
| `--z-skiplink` | 1600  | Skip navigation links        |
| `--z-toast`    | 1700  | Toast notifications          |
| `--z-tooltip`  | 1800  | Tooltips (highest)           |

## Container Usage

### Basic Containers

```css
/* Responsive container */
.container {
  width: 100%;
  margin: 0 auto;
  padding: 0 var(--space-4);
  max-width: var(--container-lg);
}

/* Different container sizes */
.container-sm {
  max-width: var(--container-sm);
}

.container-lg {
  max-width: var(--container-lg);
}

.container-xl {
  max-width: var(--container-xl);
}
```

### Responsive Container Sizing

```css
.responsive-container {
  width: 100%;
  margin: 0 auto;
  padding: 0 var(--space-4);

  /* Start with small container */
  max-width: var(--container-sm);
}

@media (min-width: 40em) {
  .responsive-container {
    max-width: var(--container-md);
    padding: 0 var(--space-6);
  }
}

@media (min-width: 64em) {
  .responsive-container {
    max-width: var(--container-lg);
    padding: 0 var(--space-8);
  }
}

@media (min-width: 80em) {
  .responsive-container {
    max-width: var(--container-xl);
  }
}
```

### Content-Specific Containers

```css
/* Reading content (optimal line length) */
.prose-container {
  max-width: var(--container-md);
  margin: 0 auto;
}

/* Wide layout for dashboards */
.dashboard-container {
  max-width: var(--container-3xl);
  margin: 0 auto;
  padding: 0 var(--space-6);
}

/* Narrow form container */
.form-container {
  max-width: var(--container-sm);
  margin: 0 auto;
  padding: var(--space-6);
}
```

```css

  display: grid;
  grid-template-columns: 1fr;
  gap: var(--space-4);
```

## Best Practices

### Container Guidelines

- **Use semantic containers** - Choose appropriate max-widths for content type
- **Maintain consistent padding** - Use spacing tokens for container padding
- **Consider reading experience** - Use narrower containers for text content
- **Plan for growth** - Choose containers that accommodate content expansion

### Breakpoint Guidelines

- **Mobile-first approach** - Start with mobile styles, enhance for larger screens
- **Use em units** - Breakpoints respect user font size preferences
- **Test at boundaries** - Ensure layouts work at breakpoint edges
- **Consider content** - Let content determine breakpoints, not devices

### Z-Index Guidelines

- **Use the scale** - Stick to predefined z-index values
- **Group by function** - Related elements should use similar z-index values
- **Avoid conflicts** - Plan layering hierarchy before implementation
- **Document usage** - Comment complex z-index relationships

### Accessibility Considerations

- **Focus management** - Ensure focus indicators are visible above overlays
- **Skip links** - Use highest z-index for skip navigation
- **Reading order** - Maintain logical DOM order regardless of visual layering
- **Screen readers** - Use proper ARIA attributes for layered content

This layout system provides a solid foundation for building responsive, accessible, and maintainable layouts across all screen sizes.
